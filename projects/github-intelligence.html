GitHub Intelligence Dashboard

Production ETL pipeline analyzing open-source investment strategy across Big 5 tech companies

Executive Summary

Built a production-grade daily ETL pipeline that processes GitHub data for 40+ repositories across 5 companies, derives 35+ metrics per repository, and powers 4 specialized Power BI dashboards. The system is fully automated via GitHub Actions, processes 2,000+ records/day, and maintains 99.9% SLA uptime while delivering <15 minute end-to-end latency.

The Business Problem

📍 Fragmented Data: Understanding which technologies Big 5 companies invest in required manually searching across 40+ GitHub repositories, with no unified view or historical tracking.

📍 Manual Analysis: Tracking metrics like star growth, language adoption, and innovation velocity required error-prone spreadsheets updated monthly.

📍 Missed Insights: Without automated analysis, stakeholders couldn't identify emerging technology trends or competitive positioning in real-time.

Daily Automation

Development Setup
VS Code + BigQuery extension

Automation: GitHub Actions triggers daily at 2:00 AM UTC. Python script extracts data, validates quality, transforms into 35+ metrics, then upserts to BigQuery. Zero manual intervention. Maintains 1-year rolling history.

Environment: Python 3.10+, PyGithub for API access, pandas for data transformation, google-cloud-bigquery client. API tokens in GitHub Secrets. Read-only access. Rate limiting: 5K requests/hour.
